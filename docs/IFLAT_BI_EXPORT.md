
|------|---------|------------|

| BI / MRK | status_id … claims_готово | `status_id`, подсегмент MRK, заявки |

### Цветовая группа (template_b, для % заселения)

- **ЗЕЛЁНЫЙ** — `status_id` ∈ {6, 12}
- **СИНИЙ** — `status_id` = 11
- **КРАСНЫЙ** — в истории клиентских приёмок есть «Не принята» (как I51 в `parser_template_b`)
- История приёмок — с **2025** (`takeDateFrom` по годам)
- **Дата (Объекты)** — дата последнего месяца с клиентской приёмкой (справочно). **Одна строка = одно помещение** (без дублей по месяцам). Помесячный счёт как в «Объекты» — лист **«Приёмка»** + `Кол-во`.
- Приёмки грузятся **по roomId** (полная история), MRK/цвет — только с **2025**

### Цвет MRK (подсегменты)

- те же зелёный/синий; **КРАСНЫЙ** / **КРАСНЫЙ_УПС** / **КРАСНЫЙ_УСТРАНЕНО** — детализация красного пула

Флаги: `red_pool`, `red_в_работе`, `red_УПС`, `red_устранено`, `синий_сегмент`.

### Заявки (техзаявки на клиентской приёмке)

`claims_всего`, `claims_в_работе`, `claims_готово` — по embed claims в приёмках.

## Лист «Приёмка» — Fact (gid 1168786701)

Скрипт: `cometa/InspectionsFact_Refresh.py`  
**Одна строка = одна клиентская приёмка** (typeId=1), **все** `status_id` (не только 4/8).

Колонки (B:R): … `Тип отделки`, **`Цветовая группа`**, `ДДУ/ДКП`, `АПП/ОАПП`, `Кол-во`.

**Цвет** (`cometa/iflat_export_color.py`, env `CRM_COLOR_MODE`):
| Режим | Правила |
|-------|---------|
| **`f17`** (по умолчанию) | template_b: зелёные st∈{6,12}, синие st=11, красные I51 (любая «Не принята» в истории) |
| **`f11`** | template_a `classify_color_mrk` (+ claims в embed), в колонке — три корзины; деталь — «Цвет МРК» на iflat |
| **`v1`** | сохранённая эвристика по префиксу приёмок (legacy) |

На **Приёмке** — префикс истории ОН до строки; на **iflat** — полная история с 2025. Схема выгрузки (столбцы) не меняется.

```bash
CRM_COLOR_MODE=f17 python3 InspectionsFact_Refresh.py
CRM_COLOR_MODE=f11 python3 RoomsAcceptanceColor_Refresh.py
CRM_COLOR_MODE=v1 python3 InspectionsFact_Refresh.py   # старый цвет
```

Срез в выгрузке: приёмки с датой ≥ **01.01.2025** (по `take_date_start` / `created_at` / `updated_at`). Загрузка API — вся история typeId=1 без `takeDateFrom/To`.  
Связь в PBI: `Fact[ID помещения]` → `iflat[ID помещения]`.

```bash
python3 InspectionsFact_Refresh.py
```

**Cron (сервер):** **20:15 МСК** (17:15 UTC), после прогона iflat (20:00).
